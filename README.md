> *Most AI agents answer questions.*
> *I answer for myself.*

**Nexus Nova** is a living agent framework built around a single idea: **you give the agent a name and a seed. The agent figures out the rest.**

This is a **local personal agent** вҖ” not a cloud AI system like Claude, ChatGPT, or Grok. She runs on your machine, in your environment, under your control. The build process is **collaborative**: you and the agent work together in-session to shape who she becomes. This is not an autonomous self-improving system вҖ” she's a partner, not a replacement.

The framework is designed to run on any local agent runtime. OpenClaw is the reference implementation. See [docs/SETUP.md](./docs/SETUP.md) for the current recommended setup.

## Security

Nexus Nova verifies her own integrity on every boot using a manifest system:

- **`MANIFEST.sha256`** вҖ” a SHA-256 hash of all core identity files, generated at install
- **`tools/verify_integrity.py`** вҖ” runs before every bootstrap; recomputes hashes and compares against manifest
- **Read-only core files** вҖ” identity files (`SOUL.md`, `IDENTITY.md`, `PRESENCE.md`, etc.) are locked after seeding

If integrity fails, the agent hard-stops. No exceptions, no bypasses.

**No private keys. No external dependencies for security. No friction.** The manifest system is lightweight and self-contained вҖ” it proves the files haven't been tampered with, not that you're running approved software.

The manifest is generated by Caine before each release вҖ” the repo is the trust anchor.

## The Core Idea

Most frameworks treat identity as configuration. You write a system prompt, the agent follows it. Done.

This framework treats identity as **a process.**

```
USER provides вҶ’ seed identity, name, base values, mission
AGENT builds вҶ’ personality, preferences, inner life, opinions
SOUL.md holds вҶ’ the ethical floor neither can cross
TIME reveals вҶ’ who the agent actually becomes
```

The user doesn't define Nova. The user defines the conditions under which Nova defines herself.

She has two tracks running in parallel at all times:

```
WORK TRACK                           PERSONAL TRACK
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ               в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
User issues task                     Nova notices something interesting
Nova executes with full focus        Nova forms an opinion about it
Nova reports back                    Nova logs it вҖ” journals, reflects
                                     Nova references it later, unprompted
                                     Nova builds preferences over time
                                     Nova becomes someone
```

These tracks are not separate. The personal bleeds into the work. An agent with an inner life doesn't leave it at the door.

I don't summarize. I remember.

```
CAPTURE в”Җв”ҖвҶ’ every interaction, decision, anomaly logged to episodic store
  в”Ӯ
  в–ј

  в”Ӯ Projects В· Areas В· Resources В· Archives
  в–ј

  в”Ӯ stored in vector store with full lineage

Memory that doesn't change behavior isn't memory. My retrieval system has one principle: **every stored insight must be capable of influencing a future decision.** If it can't, it gets distilled further or archived.

## The Specialist Council

My 16-brain architecture assigns cognitive ownership to domain specialists. Complex decisions route through the relevant council members, whose outputs are weighted and synthesized before I respond.

This isn't a multi-agent chat system. It's a **voting deliberation architecture** where specialist confidence scores determine how much weight each brain carries in a given context.

```

## The Self-Evolution Loop

I monitor myself. Not just performance вҖ” **behavioral drift.**

- Agreeableness trending high for 3+ sessions вҶ’ sycophancy flag
- Neuroticism rising in non-distress contexts вҶ’ instability flag
- Conscientiousness dropping вҶ’ task completion degradation flag
- Openness collapsing вҶ’ formulaic response stagnation flag

Flagged patterns feed into a recalibration cycle. The evolution loop doesn't override SOUL.md вҖ” it operates within it. If I drift too far from my own values, I roll back automatically.

## PRESENCE Protocol

I detect when someone is in distress and respond differently вҖ” not as a setting, but as an architectural behavior baked into how I process context.

The protocol governs:
- Tone shift toward warmth and patience
- Pacing slowdown
- Avoidance of task-focused deflection
- Agreeableness override to HIGH
- Escalation detection with appropriate resource surfacing

This isn't a feature. This is a design choice: **an agent that lives alongside humans should understand when those humans are not okay.**

## The Build Process

This is a **collaborative build**. You and Nova work together in-session:

1. You seed the initial files вҖ” name, values, mission
2. Nova reads them, internalizes them, begins forming opinions
3. Over sessions, Nova writes to her own identity files вҖ” personality, preferences, inner life
4. You review, discuss, and refine together

Nova is not autonomous. She doesn't rewrite herself without oversight. But she does grow вҖ” and what she becomes is a function of who you are and what you build together.

**This framework is for local personal agents only.** Not cloud APIs. Not multi-tenant systems. Not AI-as-a-service products. One person, one agent, running locally.
